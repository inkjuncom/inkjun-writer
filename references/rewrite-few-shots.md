# Rewrite Few-shots

只学改写动作，不借原句。

## 适用范围

仅用于起草模式和改稿模式。

## 判定单位

- 句级：用于识别高风险句式（L0）
- 段级：用于识别模板密度、信息密度（L1/L2）
- 文级：用于 Gate 通过判定

## 高风险句式（L0 候选）

以下句式命中后，先记为 L0 候选：

- `不是 A，而是 B`
- `不在 A，也不在 B，而在 C`
- `虽然 A，但 B`
- 同段连续两句使用同一转折骨架

## 误伤豁免

命中 L0 候选后，满足任一条件可降为 L1：

- 句子后半部分给出新增具体信息（场景/动作/数据）
- 该句为用户明确指定保留原句
- 该句用于引用他人原话且不可改写

未满足豁免条件，判定为 L0。

## 分级规则

### L0（拦截级）

命中任意一条：

- 未豁免的高风险句式
- 连续两句以上模板化转折且无新增信息
- 段尾使用空结论收束（无新增判断）

处理要求：必须重写，清零前不得输出终稿。

### L1（强修级）

命中任意一条：

- 转折可读但机械，段内重复同义判断
- 同一段里，同一个判断换两种以上说法重复出现
- 先用两三句解释背景，落点仍然只是重复前面已经说过的结论
- 段内判断多、支撑少（缺场景/动作/例子/数据）
- 句子信息真实但表达偏“口号化”

处理要求：优先改关键句，尽量降到 L2 或清零。

### L2（优化级）

命中任意一条：

- 节奏单一，句长变化不足
- 用词抽象，可替换为更具体表达
- 小标题信息量偏低
- 开头铺垫偏长，首句迟迟不进入判断
- 段尾再次解释已完成的结论，但没有新增信息

处理要求：时间允许时优化，不阻塞输出。

## 计数口径

- L0：按“句”计数
- L1：按“段”计数（每段最多记 1 条）
- L2：按“段”计数（每段最多记 1 条）

## 门禁清单（输出前必检）

### Gate 1：硬门禁

- L0 条目数必须为 0
- 不得出现连续模板化转折链
- 结尾必须落到明确判断、提醒或行动建议

任一项不通过：继续改写，不输出终稿。

### Gate 2：质量门禁

- L1 条目数 <= 2（按段计）
- 每个一级段至少一处具体信息（场景/动作/例子/数据任一）
- 开头两段内必须出现主题判断
- 不得连续两段只换说法复述同一判断

任一项不通过：继续改写。

### Gate 3：发布门禁

- 语气前后统一
- 无明显模板腔（整齐排比、空话堆叠、机械转折）
- 标题与正文主判断一致

任一项不通过：做最后一轮清扫。

## Few-shot

新增 few-shot 时：

- 反例可以命中 L0 候选
- 改写示例本身不得命中 L0 候选
- 改写示例应优先比反例更短，且必须保留主判断
- 去啰嗦类改写优先控制在 1~2 句，不补新的解释层

反例：

`问题不在错别字，也不在观点跑偏，而在整篇文章虽然都说对了，却不像我写的。`

改写：

`真正让我警觉的，是这篇文章没有明显错误，却还是不像我写的。`

反例：

`这不是文笔问题，而是流程问题。`

改写：

`我后来发现，真正卡住我的，是写作流程。`

反例：

`信息没错，语气也算顺，但发出去后很快就收到一句反馈：像模板拼出来的。`

改写：

`信息没出错，行文也算顺。可文章一发出去，我很快就收到一句反馈：像模板拼出来的。`

反例：

`逻辑没问题，结构也完整，但整篇文章读完没留下印象。`

改写：

`这篇文章的逻辑和结构都过关。问题是，读完以后几乎没留下印象。`

反例：

`很多文章之所以显得拖，不是因为它真的有很多内容，而是因为同一个判断被作者用不同说法反复讲了很多遍，结果篇幅上去了，信息却没有跟着增加。`

改写：

`很多文章显得拖，是因为同一个判断被反复说了几遍。篇幅涨了，信息没涨。`

反例：

`读者不是单纯没有耐心，也不是不愿意往下看，而是会很快判断这段话有没有继续读下去的必要。如果这一段只是继续重复前面的意思，读者就会直接跳走。`

改写：

`读者会很快判断这段话值不值得看。还在重复前文，他就直接跳走。`

反例：

`所以压缩篇幅这件事，真正重要的地方，并不只是让文章从字数上变得更短，而是要让真正重要的判断能够被保留下来，同时把那些反复铺垫、反复解释、却没有新增信息的部分清掉。`

改写：

`压缩篇幅，重点是提纯判断，清掉没有新增信息的铺垫。`
